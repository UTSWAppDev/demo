Remote Exercise

Which is your team?
// Open Git Bash
// Check your current directory

: pwd

// List the files and directories within your current directory

: ls

// Change directories to get to a directory you recognize, one you can easily find later.
// The command to change directories is: cd NameOfDirectory
// You can jump multiple directories by separating them with a forward slash "/"

// An example for Windows users:
: cd /c/Users/S123456/Documents

// An example for MacOS users:
: cd /Users/YourUserName

// Clone your team's repository

: git clone https://github.com/UTSWAppDev/YourTeamName
: cd YourTeamName

// Now you'll need to coordinate the next steps with your teammates. Have one team member modify the index.html
// in her local directory. After making a modification, that team member will need to commit the change to her
// local main branch.

: git commit -am "Description of your modification here"

// Now that team member's main branch needs to be pushed to the remote.

: git push origin main

// Now check index.html in your GitHub to verify that the changes have been uploaded to the shared repository

// Now have another team member modify his local branch as well. Modify index.html in another way. Perhaps
// change some style information about bg colors or text colors. Then have that team member commit the changes
// to his local branch.

: git commit -am "Description of your modification here"

// Now that team member should attempt to push the changes to the remote.

: git push origin main

// Note that this will fail due to the remote branch now having updates that the pushing user does not have.
// The solution is to fetch the remote branch's new commit data and then merge this with the commit you
// are attempting to push.

: git fetch origin

// Now your remote tracking branch "origin/main" reflects the commit made by the first team member. You can now
// merge "origin/main" into your local main as if it were a local branch. Remember the syntax for merging:

: git merge origin/main

// Resolve conflicts, if there are any. Once the merge is complete, your local file will now reflect both your
// changes as well as your teammates earlier changes. Now you are able to push your local branch to the remote.

: git push origin main

// Repeat these steps of modifying content, commiting, and pushing for the remaining two team members until
// everyone has modified the file.

// the main repository.
